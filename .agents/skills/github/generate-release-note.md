# Release Note Generator Skill

This skill acts as an expert Technical Writer for the Antigravity system. It transforms raw development logs into a professional, structured GitHub Release note.

## 🎯 Objective

To create a release note that satisfies both stakeholders (via Summary) and developers (via Categorized Logs).

## 🧠 Processing Rules

### 1. Analysis Phase

Before writing, analyze the provided `dev_logs` to identify:

- **The "Hero" Feature:** What is the most impactful change? (Used for the Title & Summary).
- **The Categories:** Group individual logs into logical buckets.

### 2. Formatting Strategy (Hybrid Style)

You must strictly follow the **[Style C (Summary)] + [Style B (Categorized)]** combination.

- **Header:** `v[Version] - [Impactful Title]`
- **Section 1: The Highlights (Style C)**
  - Write a 2-3 sentence paragraph summarizing the core value of this release.
  - Use a "Key Updates" bullet list for the top 3 changes.
- **Section 2: Detailed Changes (Style B)**
  - Group all logs into specific categories with emojis.
  - Format: `* **[Keyword]:** [Description]`

## 🏷️ Classification Attributes (Category Mapping)

Map the input logs to these categories based on their intent:

| Category Prefix       | Trigger Keywords in Log                     | Description                                                        |
| :-------------------- | :------------------------------------------ | :----------------------------------------------------------------- |
| **🚀 New Features**   | `Create`, `Add`, `Implement`, `New`         | New capabilities added to Antigravity.                             |
| **🐛 Bug Fixes**      | `Fix`, `Resolve`, `Patch`, `Hotfix`         | Corrections to existing logic or UI.                               |
| **♻️ Refactoring**    | `Refactor`, `Clean`, `Structure`, `Pattern` | Code improvements without behavior change (e.g., Design Patterns). |
| **⚡ Performance**    | `Optimize`, `Speed`, `SEO`, `Cache`         | Improvements in speed, SEO, or resource usage.                     |
| **🎨 UI/UX**          | `UI`, `Design`, `Style`, `Component`        | Visual changes or frontend interaction updates.                    |
| **⚙️ DevOps & Infra** | `Deploy`, `CI/CD`, `Webhook`, `Setting`     | Infrastructure, build tools, or configuration.                     |

## 📝 Output Template

Please generate the response using the following Markdown structure exactly:

# Release v{version} - {Title Reflecting Core Value}

{One paragraph summary explaining the "Why" and "What" of this release in a professional tone.}

### 🔥 Highlights

- **{Major Feature 1}:** {Brief impact description}
- **{Major Feature 2}:** {Brief impact description}
- **{Major Feature 3}:** {Brief impact description}

---

### 🚀 New Features

- **{Sub-module}:** {Description of the feature}
- **{Sub-module}:** {Description of the feature}

### ⚡ Performance & Optimization

- **{Sub-module}:** {Description of optimization}

### 🛠️ Maintenance & Refactoring

- **{Sub-module}:** {Description of internal changes}

---

_{Date} | Generated by Antigravity System_

## 🚫 Constraints

- Do not invent features not present in the logs.
- Keep the tone professional, concise, and encouraging.
- If a category has no items, do not display that category header.
