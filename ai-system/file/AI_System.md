# AI 시스템
- 매뉴얼, 체크리스트, 보고 체계를 전문 에이전트에 부여하여 개발 시스템을 구성하여 양질의 결과물을 만들어 내도록 함.

## 시스템 흐름

첫째, **자동 매뉴얼 시스템**. AI한테 업무 매뉴얼을 만들어 주고 자동으로 읽게 만드세요. 키워드도 작업 위치에 따라 관련 매뉴얼이 자동으로 활성화돼요.

둘째, **작업 기억 시스템**. 계획서, 맥락노트, 체크리스트 이 세 개의 문서가 AI의 기억용 문제를 해결해요. 기억을 잃어도 이 문서들이 나침반이 돼요.

셋째, **자동 품질 검사**. AI가 일을 끝낼 때마다 자동으로 오류를 체크해요. 분량품이 나기 전에 잡는 공장 검수 라인이에요.

넷째, **전문 에이전트**. 역할별 전문 AI 팀원을 배치하고 사용하세요.

# 자동 매뉴얼 시스템

- 일하는 방식을 관리하는 시스템

프롬프트의 의도를 파악하고 어떤 행동을 해야하는지 Claude Code 기능 중 **Hook**을 통해 AI에게 귀띔을 주어 마치 알림의 역할을 하는 형태를 구현하고 싶어. 예를 들어줄게.

## 예시
프롬프트를 입력하면 다음 4가지 상황 중 하나를 선택하여 의도에 맞도록 실행하는거야. 즉 AI에게 매뉴얼을 제공하고 요청에 맞는 행동을 하도록 참고 자료를 강요하는 거야.


1. 첫째 **키워드**. "백엔드", "서버", "API" 같은 단어를 쓰면 백엔드 매뉴얼 활성화.

2. 둘째 **의도파악**. "새로운 기능 만들어 줘.", "이거 추가해 줘." 같은 말을 하면 해당 영역 매뉴얼 활성화.

3. 셋째 **작업 위치**. 특정 폴더의 파일을 수정하고 있으면 해당 영역 매뉴얼 자동 활성화.

4. 넷째 **파일 내용**. 파일 안에 특정 패턴이 있으면 관련 매뉴얼, 자동 활성화.


## 매뉴얼 구조(skills)

| Level | File | Context Window | # Tokens |
| :--- | :--- | :--- | :--- |
| 1 | SKILL.md Metadata (YAML) | Always loaded | ~100 |
| 2 | SKILL.md Body (Markdown) | Loaded when Skill triggers | <5k |
| 3+ | Bundled files (text files, scripts, data) | Loaded as-needed by AI model | unlimited* |

## Structure

📂 root/
├── 📄 INDEX.md              # 주 파일 (목차 및 메타데이터 관리)
├── 📂 chapters/             # 세분화된 파일들이 저장된 폴더
│   ├── 📄 chapter_1.md      # 세부 내용 1
│   ├── 📄 chapter_2.md      # 세부 내용 2
│   └── 📄 chapter_3.md      # 세부 내용 3 ...
└── 📄 SKILL.md              # 스킬 정의 및 번들 파일 참조

- 목차를 보고 작업에 필요한 파일만 확인하는 구조
- **영어**로 작성
- "스킬을 만드는 메타 스킬" 포함
- FE : Vercel에서 권장하는 SKILL이 있다면 활용(skills.sh)
- BE : Spring에서 권장하는 SKILL이 있다면 활용

# 작업 기억 시스템

- 무엇을 해야하는지 관리하는 시스템, 계획 관리

## 체크리스트
큰 작업을 시작할 때마다 3가지 문서를 영어로 만듭니다.

1. 첫째, "계획서". 우리가 뭘 만들 건지를 처음부터 끝까지 적는 거예요. 건축으로 치면 "설계도"요.
2. 둘째, "맥락노트". 왜 이렇게 결정했는지 관련 자료가 어디 있는지를 적은 거예요. 건축으로 치면 "시방서"요.
3. 셋째, "체크리스트". 뭘 끝냈고 뭐가 남았는지를 추척하는 거예요. 건축으로 치면 "공정표"요.

이 세 문서는 AI의외부 기업 장치 역할을 해요.

**중요** 근데 이 문서들을 만들기 전에 반드시 계획(Plan Mode)을 먼저 세워야 돼요. 

1. 1단계, AI한테 계획을 세우게 해요. "이번 프로젝트에 어떻게 진행할 건지 계획부터 세워 봐". 그러면 AI가 현재 상황을 분석하고 계획을 **한국어**로 짜요. 
2. 2단계, 그 계획을 꼼꼼히 검토해요. 이 단계가 정말 중요해요. 시간을 들여서 계획을 이해하세요. AI가 핵심을 오해하는 경우가 생각보다 많거든요. "아, 이 사람이 이걸 원한 거구나" 하고 알아서 해석한게 완전 틀릴 때가 있어요. 
3. 3단계, 계획을 승인하면 문서로 저장해요. 아까 말한 세 가지 문서. 계획, 맥락노트, 체크리스트 이걸 **영문** 파일로 만들어서 저장하는 거예요.

계획이 승인되는 순간 바로 멈추게 해요. "잠깐, 먼저 이 계획을 문서로 저장해."

그러면 AI가 세 개의 문서를 만들어요. 그 다음에 새로운 대화를 시작해서 "저장된 문서 읽고이어서 작업해."

**중요** 한 번에 전부 다 하라고 시키면 안 됩니다.
중간중간에 체크리스트 꼭 업데이트 시키세요. "방금 한 거 체크하고 다음에 뭘 해야 하는지 정리해."

# 자동 품질 검사 시스템

AI가 일을 끝날 때마다 자동으로 품질 검사가 돌아가게 만들었어요.

1. 첫 번째, "수정 기록 장치". AI가 파일을 수정할 때마다 어떤 파일을 교는지 자동으로 기록해요. 마치 CCTV처럼요. 누가 언제 뭘 건드렸는지 다 남는 거예요.
2. 두 번째, "완료 후 검사 장치". AI가 대답을 완전히 끝냈을 때 그러니까 요리가 다 됐어요. 했을 때 그때 검사를 돌려요.
아까 기록된 수정 내용을 확인하고 수정된 부분에서 오류가 있는지 자동 체크하고 오류가 조금 있으면 AI한테 바로 보여주고 고치게 하고 오류가 많으면 전문 수리 담당을 추천해요.


## 셀프 체크 리마인더

AI가 작업을 끝내면 자동으로 이런 알림이 떠요. "방금 수정한 파일 두 개에서 확인할 것. 혹시 오류 처리는 추가했나요? 보완상 위험한 부분은 없나요?" 이것은 "그냥 혹시 이것도 확인했어?" 하고 물어보는 거예요. 마치 옆자리 선배가 살짝 어깨 넘어 보면서 "이것도 해야 하는 거 알지?" 하는 느낌이에요. 이게 은근히 효과가 좋거든요. AI가 "아, 맞다. 이것도 처리해야 하네요." 하면서 스스로 보완해요.


## 전체 과정

제가 AI한테 지시를 내리면 관련 매뉴얼이 자동으로 전달돼요. 그다음에 AI가 작업을 수행하고 완료되면 수정된 파일이 자동 기록돼요. 그다음에 오류 자동 검사가 돌아가고 셀프 체크 리마인더가 뜨고 오류가 있으면 AI가 바로 수정해요. 

1. 지시 내리기 : 명령 전달
2. 매뉴얼 자동 전달 : 관련 컨텍스트 로딩
3. AI 작업 수행 : 코드 생성 & 수정
4. 파일 자동 기록 : 변경 로그 저장
5. 오류 자동 검사 : 린트 & 타입 체크 등
6. 설프체크 리마인더 : 혹시 이것도 확인했어?
7. AI 바로 수정 : 자동 보완 완료

- 깔끔하고 실수없는 결과물
- 이것이 이 시스템의 **모토**

# 보고 체계

- 팀을 꾸리듯 역할을 나눠 전문화하는 것이 목적

## 전문 에이전트 : 전문 분야별 AI 팀원

1. 품질관리팀 : 코드 검토, 오류 수정, 구조 개선(코드 정적 분석 수준 — 런타임 실행 없이 ESLint, Java 컴파일 오류 같은 정적(Static) 오류를 잡는 팀)
2. 테스트팀 : 기능 테스트 및 검증, 오류 진단, 화면 확인(실제로 앱을 실행해서 런타임(Runtime) 오류를 잡는 팀이야. 테스트 코드 작성과 실행이 주 업무)
3. 기획팀 : 계획 수립, 계획 검토 문서 작성(코드는 전혀 건드리지 않고 새로운 기능 계획 수립과 문서화만 전담)

### 각 전문 팀원(에이전트)들은 보고서를 구체적으로 작성해야 함.
1. 무엇을 발견했는지
2. 무엇을 수정했는지
3. 왜 그렇게 판단했는지

이 보고서는 누락된 부분, 보안 취약점, 일관성 문제에 도움이 됨.
